# 门店筹建进度管理系统 V3

## 概述

生成完整的门店筹建进度管理系统 HTML 文件，包含所有配置和数据。

## 使用方式

当用户需要创建或恢复门店筹建进度管理系统时，使用此 skill 生成完整的 HTML 文件。

### 生成命令

```bash
python3 ~/.workbuddy/skills/store-progress-manager-v3/scripts/generate.py --output <输出路径>
```

## 配置说明

### 1. 阶段节点配置 (PHASES)

系统包含 7 个阶段，共 39 个节点：

| 阶段 | 简称 | 节点数 | 颜色 |
|------|------|--------|------|
| 选址阶段 | 选址 | 2 | #7c3aed |
| 设计阶段 | 设计 | 5 | #2563eb |
| 施工前准备阶段 | 施工前准备 | 7 | #0891b2 |
| 施工阶段 | 施工 | 15 | #d97706 |
| 人员阶段 | 人员 | 2 | #059669 |
| 整改阶段 | 整改 | 2 | #7c3aed |
| 开业阶段 | 开业 | 6 | #16a34a |

### 2. 基本信息字段配置 (BASIC_FIELDS)

默认包含以下自定义字段：
- 门店类型（单选：直营店,加盟店）
- 装修形式（单选：自装,公装）
- 工程负责人（单选：夏博谦,曹飞,罗增）
- 施工队伍名称（单选：北京大助,深圳罗学栋,深圳丁海,深圳群卓,豪,南京蔡健,上海刘裕银,上海许飞跃,暂无）
- 门店面积（文本）

### 3. 数据看板配置

支持的数据看板组件：
- 筹建中门店状态统计
- 门店类型分布
- 区域分布
- 工程负责人工作量
- 施工队伍使用情况
- 筹建周期统计
- 预估开业门店时间

### 4. 区域颜色配置

同一区域的门店显示相同颜色：
- 苏州：蓝色系
- 深圳：绿色系
- 东莞：橙色系
- 泰州：玫红色系
- 嘉兴：紫色系
- 南京：青色系
- 淮安：深橙色系
- 南通：天蓝色系
- 上海：金黄色系
- 马鞍山：深蓝色系
- 常州：黄绿色系
- 昆山：红色系
- 张家港：深紫色系
- 靖江：深红色系

### 5. 示例门店数据

包含 32 家示例门店，涵盖不同区域、不同状态（施工中、设计中、测量中等）。

## 文件结构

```
store-progress-manager-v3/
├── SKILL.md                 # 本文件
├── scripts/
│   └── generate.py          # HTML 生成脚本
└── assets/
    └── template.html        # HTML 模板（可选）
```

## 生成参数

```bash
python3 scripts/generate.py --output /path/to/output.html
```

参数说明：
- `--output`: 输出文件路径（默认: 进度管理系统.html）

## 数据持久化

生成的 HTML 系统使用 localStorage 保存：
- `storePhases`: 阶段节点配置
- `storeData`: 门店数据
- `basicFields`: 基本信息字段配置
- `dashboardWidgets`: 数据看板组件配置
- `storePhasesVersion`: 配置版本号

## 功能特性

1. **甘特图展示**: 横向时间轴展示各门店进度
2. **节点编辑**: 可添加、删除、修改阶段和节点
3. **基本信息设置**: 自定义门店基础信息字段
4. **数据看板**: 可视化统计图表
5. **导入导出**: 支持 Excel 导入导出
6. **区域颜色**: 同一区域门店标签颜色一致
7. **预估开业日历**: 按日期展示预估开业门店

## 注意事项

1. 生成的 HTML 文件是独立的单页应用，无需后端服务
2. 数据存储在浏览器 localStorage 中，清除浏览器数据会丢失
3. 建议使用"导出数据"功能定期备份
4. 支持 Chrome、Firefox、Safari 等现代浏览器
