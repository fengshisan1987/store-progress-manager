# 门店筹建进度管理系统 - 权限控制逻辑

## 角色定义

系统支持三种角色：

| 角色 | 标识 | 权限级别 |
|------|------|----------|
| 超级管理员 | `superadmin` | 最高权限，可执行所有操作 |
| 管理员 | `admin` | 中等权限，可编辑门店数据但不能修改系统设置 |
| 访客 | `guest` / `visitor` | 最低权限，只能查看数据 |

## 权限检查函数

### 核心函数

```javascript
// 获取当前用户角色
function checkRole() {
  const currentUser = JSON.parse(localStorage.getItem('auth_current_user') || '{}');
  return currentUser.role || 'visitor';
}

// 判断是否为超级管理员
function isSuperAdmin() {
  return checkRole() === 'superadmin';
}

// 判断是否为管理员（包括超管）
function isAdmin() {
  return checkRole() === 'admin';
}

// 判断是否为访客
function isVisitor() {
  return checkRole() === 'guest' || checkRole() === 'visitor';
}
```

## 权限数据存储

用户角色信息存储在 localStorage 中：

```javascript
// 当前登录用户
localStorage.getItem('auth_current_user')
// 格式: { username: '用户名', role: 'superadmin|admin|guest' }

// 所有授权账号
localStorage.getItem('auth_accounts')
// 格式: [{ username: '用户名', password: '密码', role: '角色' }, ...]
```

3. **超级管理员测试**: 使用超管账号登录，验证所有功能可用
4. **权限绕过测试**: 尝试通过浏览器控制台直接调用受限函数，验证权限检查是否生效
