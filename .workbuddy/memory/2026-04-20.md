## 修复日期选择器时区偏差问题

**问题**：用户选择 3月12日，但实际保存为 3月11日。

**根因**：`confirmDatePicker()` 函数中使用 `toISOString().split('T')[0]` 获取日期字符串，但 `toISOString()` 返回的是 UTC 时间。中国时区为 UTC+8，所以本地 3月12日 00:00 转换为 UTC 是 3月11日 16:00，截取后变成 "2026-03-11"。

**修复**：将日期格式化改为使用本地时间方法：
```javascript
const year = _datePickerSelectedDate.getFullYear();
const month = String(_datePickerSelectedDate.getMonth() + 1).padStart(2, '0');
const day = String(_datePickerSelectedDate.getDate()).padStart(2, '0');
const dateStr = `${year}-${month}-${day}`;
```

## 修复阶段配置被重置的问题

**根因**：代码中的 `PHASES_VERSION` 版本号机制会在版本号变化时强制重置阶段配置为默认值：
```javascript
if (savedPhases && savedVersion === PHASES_VERSION) {

```

**修复**：修改阶段配置加载逻辑，优先使用用户保存的数据，不再因版本号变化而强制重置：
```javascript
if (savedPhases) {
  try {
    PHASES = JSON.parse(savedPhases);
  } catch (e) {
    PHASES = JSON.parse(JSON.stringify(DEFAULT_PHASES));
  }
} else {
  PHASES = JSON.parse(JSON.stringify(DEFAULT_PHASES));
}
```

if (regionStatsRegion && regionStartDate && regionEndDate && regionStatus) {
  dashboardFilters.regionStats.region = regionStatsRegion.value || 'all';
  dashboardFilters.regionStats.startDate = regionStartDate.value || '';
  dashboardFilters.regionStats.endDate = regionEndDate.value || '';
