## 角色权限系统实现

### 权限控制逻辑
实现了三级角色权限系统：
- **超级管理员(superadmin)**: 可执行所有操作
- **管理员(admin)**: 可编辑门店数据，但不能修改系统设置
- **访客(guest)**: 仅可查看，不能编辑任何数据

### 权限检查函数
```javascript
function checkRole() - 获取当前用户角色
function isSuperAdmin() - 判断是否为超管
function isAdmin() - 判断是否为管理员
function isVisitor() - 判断是否为访客
```

### 受控功能清单
1. 门店编辑/删除按钮 - 访客不可见
2. 最新进展日期编辑 - 访客只读
3. 工程增减项备注 - 访客只读
4. 风险情况备注 - 访客只读
5. 节点点击完成 - 访客无响应
6. 门店编辑器 - 访客无法打开
7. 删除门店 - 访客无法执行
8. 节点编辑器 - 仅超管可访问
9. 基本信息设置 - 仅超管可访问
10. 导入数据 - 仅超管可访问
11. 导出数据 - 超管和管理员可访问
12. 侧边栏菜单 - 根据角色动态生成

## 修复 iOS Safari 打不开问题

**问题**：苹果手机浏览器打开进度管理系统显示白屏/无法加载

**根因**：`<script src="https://cdn.sheetjs.com/xlsx-0.20.0/package/dist/xlsx.full.min.js">` 是同步阻塞加载。在手机网络不稳定时，CDN 加载超时会导致整个页面卡死/白屏。

**修复方案**：
1. 移除 `<head>` 中的同步 `<script src="...xlsx...">` 引用
2. 新增 `loadXlsx(callback)` 异步按需加载函数
3. 仅在用户触发导入/导出时才调用 `loadXlsx()` 加载库
4. 加载失败时有友好提示，主页面不受影响

**修改文件**：`/Users/fengshisan/WorkBuddy/20260410105648/进度管理系统.html`

## 修复手机端布局问题

**问题**：iPhone Safari 打开看板时，"区域门店及类型统计"和"门店周期统计"卡片标题文字变成竖排；navbar 右侧内容被截断。

**根因**：卡片 header 内标题 + 多个筛选控件（select/date/select）在手机窄屏排在同一行，导致标题被极度压缩。

**修复方案**：
1. `@media (max-width: 768px)` 中为 `.region-stats-section-wrapper` 和 `.cycle-stats-section-wrapper` 的 `collapsible-header` 添加 `flex-wrap: wrap`，让筛选器换行到标题下方
2. 筛选控件字体和尺寸缩小适配小屏
3. `.settings-btn` 在手机端改为 `right: 12px`，避免遮挡 navbar

## 修复手机端面板无法关闭问题

**问题**：手机端打开"编辑节点"或"管理字段"面板后，点击关闭按钮或遮罩层无法关闭面板。

**根因**：
1. `#overlay` 遮罩层的 onclick 只调用了 `closePanel()` 和 `closeNodeEditor()`，**没有调用 `closeFieldManager()` 等其他面板的关闭函数**
2. 手机端关闭按钮尺寸太小（28px），点击区域不够

**修复方案**：
1. 遮罩层 onclick 添加所有面板关闭函数：`closePanel(); closeNodeEditor(); closeFieldManager(); closeStoreEditor(); closeBasicFieldSettings(); closeGroupManager();`
2. 手机端关闭按钮增大到 40px，添加 `active` 状态反馈
3. 手机端 `.detail-panel` 添加 `visibility: hidden` 和 `transition` 控制，确保关闭时完全不可见（修复 iOS Safari 面板残留显示问题）
